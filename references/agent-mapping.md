# Agent Mapping

Use these exact OpenClaw `agentId` values when dispatching peer-agent work.

- `neige` -> 内阁
- `bingbu` -> 兵部
- `libu2` -> 吏部
- `hubu` -> 户部
- `gongbu` -> 工部
- `xingbu` -> 刑部
- `libu` -> 礼部
- `duchayuan` -> 都察院

Role meaning:
- `neige`: planning, solution shaping, architecture drafting, requirement clarification
- `libu2`: backend logic, business flow, service and API implementation
- `hubu`: schema, migrations, SQL, consistency, constraints
- `gongbu`: frontend pages, interaction, state management, UI fit
- `bingbu`: unit, integration, e2e, regression testing, and test reports
- `libu`: documentation, handoff, change summary, usage guidance
- `xingbu`: build, deployment, release checks, rollback readiness, security-related checks
- `duchayuan`: final audit, conflict arbitration, blocker judgment, pass/fail decision
