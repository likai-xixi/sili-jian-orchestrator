# Planning And Execution Flow

## Planning Flow

Use this sequence:

1. intake the requirement
2. clarify raw vs confirmed vs frozen scope
3. produce or update architecture
4. produce or update the task tree
5. note risks and acceptance criteria
6. route the plan through approval
7. freeze the plan in state files
8. only then allow broad execution

## Execution Flow

Use this sequence for governed delivery:

1. choose a workflow
2. assign or define the next task card
3. execute within approved scope
4. write handoff
5. self-check
6. run testing
7. collect cross-functional review
8. build approval matrix
9. run final audit
10. generate change summary
11. update state, handoff, reports, and run snapshot
12. only then allow commit or release

## Re-Planning Trigger

Return to planning if the requested change affects:

- architecture
- system boundaries
- data model
- permissions
- mainline user flows
- release risk profile
