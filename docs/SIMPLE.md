# 简易版使用文档

这份文档适合第一次快速上手。

如果你不想一开始就看完整治理制度，只想先把技能装好、跑起来、接管一个指定项目目录，这一份就够了。

## 一、这个技能是干什么的

`sili-jian-orchestrator` 是一个面向 OpenClaw 的“司礼监 / 总调度”技能。

它不会默认直接开发，而是优先做：

- 目录识别
- 首次启用引导
- 项目接管检查
- 治理骨架初始化
- 状态、交接、测试、工作流、门禁管理

## 二、什么时候用

推荐在这些场景使用：

1. 接管一个做到一半的项目
2. 恢复一个暂停中的项目
3. 给旧项目补治理骨架
4. 把模糊需求先整理成方案
5. 把新增功能纳入既有治理体系

不推荐在这些场景使用：

1. 单纯改一个按钮
2. 修一个很小的 bug
3. 写一个不需要治理流程的小接口

## 三、最短安装思路

把整个技能目录复制到 OpenClaw 当前 workspace 的 `skills/` 下即可。

例如：

```bash
mkdir -p /home/claw/clawd/skills
rm -rf /home/claw/clawd/skills/sili-jian-orchestrator
cp -r "/mnt/c/Users/11131/Desktop/auto agent/sili-jian-orchestrator" /home/claw/clawd/skills/
```

安装后验证：

```bash
openclaw skills list | grep sili-jian-orchestrator
```

## 四、第一次怎么用

第一次不要让它直接开发，先跑首次启用引导。

推荐提示词：

```text
使用 $sili-jian-orchestrator 对当前目录执行首次启用引导。

要求：
1. 先不要直接开发
2. 先判断当前目录是技能目录、项目目录还是未知目录
3. 检查 OpenClaw 中司礼监与同级部门 agent 是否已就绪
4. 如果当前目录是项目目录，再继续检查治理状态、tests、workflows、状态文件、恢复入口
5. 输出首次引导结果，至少包含：
   - 当前目录模式
   - peer-agent 是否就绪
   - 当前环境意味着什么
   - 最安全的下一步
   - 建议我下一条输入什么提示词
6. 如果当前目录已经是项目目录，再继续输出首轮接管结果
7. 在完成上述检查前，不要进入实现阶段
```

## 五、如果要接管指定项目目录

如果你当前 agent 的 workspace 不是项目目录，而你又想接管某个明确项目路径，直接用这条：

```text
使用 $sili-jian-orchestrator 接管指定项目目录，不要把当前 workspace 根目录当成项目目录。

目标项目根目录：
<项目绝对路径>

要求：
1. 不要把当前 workspace 根目录当成业务项目根目录
2. 将 <项目绝对路径> 视为唯一目标项目目录
3. 先对该目录执行首次启用引导
4. 再执行首轮接管检查
5. 在完成检查前，不得进入实现阶段
6. 如果需要初始化治理骨架，也只能写入 <项目绝对路径>
```

## 六、简易版建议

第一次推荐分两轮走：

1. 先跑首次启用引导
2. 确认目录没认错后，再让它正式接管
