# 按推进顺序使用的提示词手册

这份文档把提示词按真实推进顺序整理，方便直接从前到后使用。

适用两类主场景：

1. 新建项目
2. 中途接管项目

---

## 一、新建项目：从 0 到 1 的提示词顺序

### 第 1 步：首次启用引导

```text
使用 $sili-jian-orchestrator 对当前目录执行首次启用引导。

要求：
1. 先不要直接开发
2. 先判断当前目录是技能目录、项目目录、workspace 根目录还是未知目录
3. 检查 OpenClaw 中司礼监、内阁、都察院、六部 agent 是否已就绪
4. 如果当前目录是项目目录，再继续检查治理状态、tests、workflows、状态文件、恢复入口
5. 输出首次引导结果，至少包含：
   - 当前目录模式
   - peer-agent 是否就绪
   - 当前环境意味着什么
   - 最安全的下一步
   - 建议我下一条输入什么提示词
6. 在完成上述检查前，不要进入实现阶段
```

### 第 2 步：新建项目首轮接管检查

```text
使用 $sili-jian-orchestrator 将当前目录作为新建项目处理，不要直接开发。

要求：
1. 将当前场景识别为 new-project
2. 检查 ai/、tests/、workflows/、docs/ 是否存在
3. 检查是否已有状态文件、交接入口、恢复机制、审批规则、测试规则、workflow 模板
4. 输出首轮接管结果，至少包含：
   - 当前项目识别结果
   - 当前场景
   - 缺失的治理文件
   - 缺失的测试体系
   - 缺失的 workflow 模板
   - 需要先创建的目录和文件
   - 当前是否具备方案阶段条件
   - 当前是否允许进入执行阶段
   - 第一轮 next_action
5. 在完成检查前，不得进入实现阶段
```

### 第 3 步：批准治理骨架初始化

```text
批准执行 bootstrap governance，但仅限治理骨架初始化，不得进入功能开发。

要求：
1. 只初始化项目治理骨架
2. 创建并补齐 ai/、tests/、workflows/、docs/ 所需结构与模板
3. 初始化状态文件、交接文档、审批规则、测试规则、workflow 模板、任务卡模板
4. 完成后输出：
   - 本次创建的目录和文件清单
   - 当前状态机阶段
   - 当前 next_action
   - 当前是否具备方案阶段条件
   - 当前是否允许进入执行阶段
5. 完成治理初始化后停止，不得直接开始实现
```

### 第 4 步：进入方案阶段

```text
使用 $sili-jian-orchestrator 进入方案阶段，不要直接开发。

要求：
1. 由内阁先补全需求并形成方案
2. 明确主链路、边界、任务树、风险、验收标准
3. 冻结 architecture.md 与 task-tree.json
4. 在方案未审批通过前，不得进入大规模执行
5. 输出：
   - 当前方案状态
   - 是否已形成冻结需求
   - 是否具备 plan-approved 条件
   - 下一步应派给哪个部门
```

### 第 5 步：进入执行前确认

```text
请确认当前项目是否已具备进入执行阶段的条件。

检查：
1. 方案是否冻结
2. architecture.md 是否完成
3. task-tree.json 是否完成
4. 审批是否通过
5. orchestrator-state.json 是否已更新到可执行状态

输出：
   - 是否允许进入执行阶段
   - 若不允许，阻塞项是什么
   - 若允许，第一批执行任务应如何派发
```

## 二、中途接管项目：从盘点到恢复推进的提示词顺序

### 第 1 步：首次启用引导

```text
使用 $sili-jian-orchestrator 对当前目录执行首次启用引导。

要求：
1. 先不要直接开发
2. 先判断当前目录是技能目录、项目目录、workspace 根目录还是未知目录
3. 检查 OpenClaw 中司礼监、内阁、都察院、六部 agent 是否已就绪
4. 如果当前目录是项目目录，再继续检查治理状态、tests、workflows、状态文件、恢复入口
5. 输出首次引导结果，至少包含：
   - 当前目录模式
   - peer-agent 是否就绪
   - 当前环境意味着什么
   - 最安全的下一步
   - 建议我下一条输入什么提示词
6. 在完成上述检查前，不要进入实现阶段
```

### 第 2 步：中途接管首轮盘点

```text
使用 $sili-jian-orchestrator 接管当前项目，场景按 mid-stream-takeover 处理，不得直接开发。

要求：
1. 执行首轮接管检查
2. 检查项目识别、ai/、tests/、workflows/、交接入口、状态文件、审批规则、测试规则、工作流模板、步骤快照、任务卡模板
3. 输出首轮接管结果，至少包含：
   - 当前项目识别结果
   - 当前属于哪种场景
   - ai/ 是否齐全
   - tests/ 是否齐全
   - workflows/ 是否齐全
   - 是否存在显式状态机
   - 是否存在最近步骤快照
   - 缺失的治理文件
   - 缺失的测试体系
   - 缺失的 workflow 模板
   - 缺失的恢复机制
   - 需要先创建的文件和目录
   - 是否具备方案阶段条件
   - 是否具备执行阶段条件
   - 是否具备测试阶段条件
   - 第一轮 next_action
   - 当前是否允许立即进入执行阶段
4. 在完成上述检查前，不得进入实现阶段
```

### 第 3 步：批准治理补齐

```text
批准执行 bootstrap governance，但仅限治理补齐，不得直接开始功能开发。

要求：
1. 只补齐缺失的治理骨架、状态文件、交接入口、测试目录、workflow 模板、报告模板
2. 对已有 ai/ 残留内容优先兼容，不要粗暴覆盖未知文件
3. 完成后输出：
   - 本次新建与补齐的目录和文件清单
   - 当前状态机阶段
   - 当前 next_action
   - 当前是否具备恢复推进条件
   - 当前是否允许进入执行阶段
4. 完成治理补齐后停止，不得直接开始实现
```

### 第 4 步：执行恢复摘要

```text
使用 $sili-jian-orchestrator 对当前项目执行 recovery summary。

要求：
1. 读取 START_HERE.md、project-meta.json、project-handoff.md、orchestrator-state.json、task-tree.json、最近报告、最近 run 快照、active handoff
2. 输出 recovery summary，至少包含：
   - 当前项目
   - 当前阶段
   - 已完成任务
   - 进行中任务
   - 阻塞任务
   - 最近方案审查结论
   - 最近结果终审结论
   - 最近测试结论
   - 当前 next_action
   - 下一步派给谁
   - 当前是否允许进入执行阶段
```

### 第 5 步：决定恢复后先做什么

```text
请基于 recovery summary 判断当前项目下一步应该优先做什么。

要求：
1. 优先判断主链路是否已明确
2. 优先判断当前最大阻塞项
3. 若缺方案，先回方案层
4. 若缺测试基线，先补测试基线
5. 若具备执行条件，再给出第一批执行任务建议

输出：
   - 当前唯一主目标
   - 当前最高优先级任务
   - 当前 blocker 级别
   - 当前 next_action
   - 建议派给哪个部门
```

## 三、指定项目目录接管：适合 workspace 与项目目录分离的情况

如果当前 agent 固定在某个 workspace，但真正要接管的业务项目在别的路径，推荐始终使用这一类提示词。

### 第 1 步：指定项目目录执行首次引导

```text
使用 $sili-jian-orchestrator 接管指定项目，不要把当前 workspace 根目录当成业务项目目录。

目标项目根目录：
<项目绝对路径>

要求：
1. 不要把当前 workspace 根目录当成业务项目根目录
2. 将 <项目绝对路径> 视为唯一目标项目目录
3. 先对 <项目绝对路径> 执行首次启用引导
4. 检查 OpenClaw 中司礼监、内阁、都察院、六部 agent 是否已就绪
5. 在完成上述检查前，不得进入实现阶段
```

### 第 2 步：指定项目目录执行首轮接管

```text
继续使用 $sili-jian-orchestrator 接管指定项目目录：
<项目绝对路径>

要求：
1. 只把 <项目绝对路径> 视为唯一目标项目目录
2. 对该目录执行首轮接管检查
3. 输出完整首轮接管结果
4. 如需治理初始化，只允许写入 <项目绝对路径>
5. 在完成检查前，不得进入实现阶段
```

### 第 3 步：指定项目目录批准治理初始化

```text
批准执行 bootstrap governance，但仅限对以下项目目录进行治理初始化：

<项目绝对路径>

要求：
1. 不允许写入当前 workspace 根目录
2. 只允许写入 <项目绝对路径>
3. 只初始化或补齐治理骨架，不得直接进入功能开发
4. 完成后输出本次创建和更新清单、当前状态、next_action 和执行权限判断
```
